## 🔧 修改执行时间

### 如需修改为其他时间

**公式**: 北京时间 = UTC + 8 小时

**示例**:

| 北京时间 | UTC 时间 | Cron 表达式 |
|---------|---------|------------|
| 06:00 | 22:00（前一天） | `0 22 * * *` |
| **07:00** | **23:00（前一天）** | **`0 23 * * *`** |
| 08:00 | 00:00 | `0 0 * * *` |
| 09:00 | 01:00 | `0 1 * * *` |
| 20:00 | 12:00 | `0 12 * * *` |

---

### 修改方法

编辑 `.github/workflows/daily-update.yml`:

```yaml
on:
  schedule:
    - cron: '分钟 小时 * * *'
```

**Cron 格式**: `分钟 小时 日期 月份 星期`

## 📝 故障排查

### 如果未按时执行

**检查项**:
1. GitHub Actions 是否启用
2. Workflow permissions 是否为 "Read and write"
3. 仓库是否有活动（长期无活动会休眠）
4. Cron 表达式是否正确

**解决方案**:
1. 手动触发一次工作流激活
2. 检查仓库设置
3. 添加健康检查工作流（已添加）

---

### 如果执行失败

**检查项**:
1. Actions 日志中的错误信息
2. Secrets 配置是否正确
3. 网络连接是否正常
4. Python 依赖是否完整

**解决方案**:
1. 查看详细日志
2. 重新配置 Secrets
3. 手动触发测试
